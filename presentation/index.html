1. HIS AND HER STORY

  • 1998 W3C froze HTML standard at 4.01 and the era of XML begun
  • Transitional phase of XHTML started and XHTML Strict was the ultimate goal
  • W3C then began work on XHTML 2.0 which would have been completely backwards incompatible
  • 2003 Guys at Opera were sure that XML was not the road to salvation
  • They began POC work extending HTML Form Tags without breaking backwards compatibility. This POC evolved to Web Forms 2.0
  • This team led by Ian Hickson soon gained momentum as Mozilla and Apple became interested

1. HIS AND HER STORY

  • 2004 WHATWG (Web Hypertext Application Technology Working Group) was formed by Apple, Mozilla and Opera
  • 2005 Hickson moved to Google to continue the work under the name Web Applications 1.0
  • 2006 W3C resurrected HTML Working Group and decided to use WHATWG spec as a foundation
  • W3C group roster included players like Sam Ruby of IBM, Chris Wilson of Microsoft and Majiec Stachowiak of Apple
  • Inventor of JavaScript and Mozilla CTO Brendan Eich, WebKit and Safari Architect David Hyatt and inventor of CSS and Opera CTO Håkon Wium Lie joined Hicksons team at WHATWG

3. NEW ELEMENTS

<header>

A group of introductory or navigational aids. A header element typically contains the section’s heading (an h1–h6 element or an hgroup element), but can also contain other content, such as a table of contents, a search form, or any relevant logos.

3. NEW ELEMENTS

<hgroup>

The hgroup element is typically used to group a set of one or more h1-h6 elements — to group, for example, a section title and an accompanying subtitle. W3C Specification

3. NEW ELEMENTS

<footer>

The footer element represents the footer for the section it applies to. A footer typically contains information about its section such as who wrote it, links to related documents, copyright data, and the like.
HOUSTON INC.

3. NEW ELEMENTS

<article>

The article element represents a component of a page that consists of a self-contained composition in a document, page, application, or site and that is intended to be independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content. W3C Specification

3. NEW ELEMENTS

<section>

The section element represents a generic document or application section…The section element is not a generic container element. When an element is needed for styling purposes or as a convenience for scripting, authors are encouraged to use the div element instead.
HOUSTON INC.

3. NEW ELEMENTS

<aside>

The aside element represents a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.
HOUSTON INC.

3. NEW ELEMENTS

<nav>

The nav element represents a section of a page that links to other pages or to parts within the page: a section with navigation links. Not all groups of links on a page need to be in a nav element only sections that consist of major navigation blocks are appropriate for the nav element. In particular, it is common for footers to have a list of links to various key parts of a site, but the footer element is more appropriate in such cases, and no nav element is necessary for those links.
HOUSTON INC.

5. CSS preprocessors

CSS preprocessors take code written in the preprocessed language and then convert that code into the same old css we’ve been writing for years. Because they aren’t css, they aren’t bound by the limitations of css.The preprocessed language can give you more functionality than css as long as it eventually makes sure everything is output as css and works in a browser like we expect.
  • Less (http://lesscss.org/)
  • Sass (http://sass-lang.com/)
  • Stylus (http://learnboost.github.com/stylus/)
  • Zuss(https://github.com/tomyeh/ZUSS)

6. Responsive Design

Responsive Web Design (RWD) essentially indicates that a web site is crafted to use W3C CSS3 media queries with fluid proportion-based grids, to adapt the layout to the viewing environment, and probably also flexible images. As a result, users across a broad range of devices and browsers will have access to a single source of content, laid out so as to be easy to read and navigate with a minimum of resizing, panning and scrolling.
